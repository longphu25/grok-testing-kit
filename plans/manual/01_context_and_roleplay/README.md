# Step 1: Initialize context (Context & Role-play)

---

## Purpose

Set up the **Senior QA Engineer** role and load the project context so that AI shapes the testing expert mindset. This step limits the AI's knowledge space, making the answer more in-depth and realistic.

## How to use

1. Open the file`prompt.txt`in this folder.
2. Copy the content below the line`---START---`.
3. Replace internal parts`[...]`with real data:
   - **Project name / feature** to test
   - **System Context** (short description of the current app)
   - **MVP goal** (testing scope this round)
   - **Attachments** (Requirements, User Stories, Figma, PDF...)
4. Paste into AI chat and send.
5. Wait for the AI ​​to confirm **"I understand the context and am ready"** before going to Step 2.

## Note

- The more detailed the document → the more accurate the AI ​​analysis.
- If there are Figma or screenshots, additional descriptions of important UI elements.
- This step only needs to be run **once** at the beginning of the conversation.
